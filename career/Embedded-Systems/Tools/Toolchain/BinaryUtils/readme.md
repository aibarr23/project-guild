# Binary Utils

Binary utilities (often known as binutils) are crucial, command-line tools in embedded development used to analyze, manipulate, and format executable code—typically ELF (Executable and Linkable Format) files—after compilation and before flashing onto a microcontroller. These utilities help manage memory, debug, and optimize binary sizes. 

## Essential GNU Binary Utilities for Embedded Systems 

### objcopy: 
- Translates object files from one format to another, commonly used to convert ELF files into raw binary (.bin) or Intel Hex (.hex) files for flashing.
### objdump: 
- Displays detailed information from object files, including disassembling machine code to assembly, analyzing section headers, and debugging.
### size: 
- Lists section sizes (text, data, bss) of an object or executable file, vital for monitoring memory consumption in resource-constrained devices.
### nm: 
- Lists symbols from object files, allowing developers to identify variable locations, function addresses, and symbol table issues.
### readelf: 
- Displays detailed information about ELF format files, including headers, sections, and dynamic linking information.
### strip: 
- Removes symbol table and debugging information from an object file to reduce binary size, often done for production firmware.
### strings: 
- Extracts printable strings from binary files, useful for finding firmware versions or debug messages.
### addr2line: 
- Converts program addresses (e.g., from a crash stack trace) back into file names and line numbers, aiding in debugging. 


## Common Use Cases
### Memory Analysis: 
- Using size and nm to ensure firmware fits within restricted flash and RAM limits.
### Firmware Formatting: 
- Using objcopy to convert ELF files to .hex or .bin formats for loaders.
### Debugging/Disassembly: 
- Using objdump to inspect assembly code generated by the compiler to troubleshoot issues.
### Post-processing: 
- Using strip to minimize final binary image size before deployment.
